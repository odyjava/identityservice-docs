WHAT這是什麼服務？

把「登入」這件事從每個產品中抽出來，集中做對、做安全。

🎯

唯一的認證權威

帳號、密碼、MFA、Token 簽發與撤銷全部集中在這裡。各產品系統不碰密碼，只驗證這裡簽發的 Token。

🏢

平台級多租戶

每個接入的產品是一個獨立「平台」（realm），帳號、簽章金鑰、政策完全隔離——A 平台的 Token 在 B 平台一定無效。

🧩

平台內再分租戶

平台之內還能有自己的命名空間（tenant，例如商城裡的各商家），同名帳號在不同租戶間互不衝突。

📜

凍結契約驅動

API 依消費方凍結的 OpenAPI 契約實作，並以黑箱契約測試持續守護——行為就是文件，文件就是行為。

⚡

Serverless 架構

以 AWS Lambda 承載、按用量計費、自動擴縮，平時近乎零成本，尖峰不用擔心容量。

🌐

標準 JWT 生態

簽發 RS256 JWT，公鑰經 JWKS 端點公開，任何服務都能本地驗簽，不需回呼登入服務。

ARCH整體架構

請求從使用者裝置出發，經過邊緣網路與 API 閘道，由無伺服器函式處理，資料落在關聯式資料庫。

FLOW登入流程

一次登入在幕後發生的事——每一步都有安全檢查，任何一關不過就停止。

解析平台身分

先確認請求屬於哪個平台（網域對映或 header），未知或停權的平台直接拒絕——連帳號驗證都不會開始。

AUTH_PLATFORM_UNKNOWN

驗證帳號密碼

密碼以 Argon2id 雜湊比對（抗 GPU 暴力破解）。「帳號不存在」與「密碼錯誤」回應完全相同，外部無法探測哪些帳號存在。

AUTH_INVALID_CREDENTIALS

檢查帳號狀態與鎖定

停用的帳號一律拒絕；連續失敗達門檻會暫時鎖定一段時間，即使之後輸入正確密碼也要等鎖定解除。

AUTH_ACCOUNT_LOCKED

分流：是否啟用 MFA？

依帳號的多因子驗證設定走向不同結局。

未啟用 → 直接發 Token

簽發短效 Access Token（JWT）與長效 Refresh Token，登入完成。

已啟用 → 回傳挑戰票

先不發 Token，回一張 5 分鐘、一次性的 mfa_token；輸入手機驗證器的 6 位數（或備援碼）通過後才簽發 Token。

TOKENToken 設計與輪替

雙 Token 制：短效 Access 負責存取、長效 Refresh 負責續期——而 Refresh 每用一次就換一張。

🎫

Access Token（JWT）

效期以「分鐘」計（一般會員較長、管理者更短）。內含簽發者、平台、帳號、租戶與類型等聲明，以 RS256 私鑰簽章；任何服務用公開的 JWKS 公鑰即可本地驗證，偽造與竄改必被識破。

🔄

Refresh Token（不透明）

約 30 天滑動效期的隨機字串，伺服器只儲存它的加鹽雜湊——就算資料庫外洩也拿不到可用的 Token。它是唯一的續期機制，且每次使用即輪替作廢。

RULES安全規則

這些規則寫死在程式與測試裡，不是紙上原則。

密碼永不明文儲存

採 Argon2id（記憶體硬化雜湊）搭配隨機鹽值；伺服器也「不知道」你的密碼。

帳號不可枚舉

登入失敗不透露帳號是否存在；登出對不存在的 Token 也回成功，不給攻擊者任何探測訊號。

暴力破解防護

連續失敗自動觸發暫時鎖定；解鎖前正確密碼也無法登入。

Token 竊用即斷

已輪替的 Refresh Token 再次出現，視為憑證外洩：整個帳號的 Refresh 全部撤銷、尚未過期的 Access 進黑名單。

機敏資料加密落地

簽章私鑰與 MFA 密鑰以 AES-256-GCM 加密後才寫入資料庫；Refresh Token 與備援碼只存雜湊。

一次性憑證

MFA 挑戰票（5 分鐘）與備援碼皆一次性，用過即廢；備援碼明碼只在啟用當下顯示一次。

平台完全隔離

每平台獨立簽章金鑰與帳號空間；系統對系統的 API Key 也綁定單一平台，跨平台操作一律拒絕。

嚴格輸入驗證

所有請求欄位嚴格檢查，未知欄位直接拒絕；錯誤訊息絕不洩漏內部細節，僅附追蹤碼供查詢。

MFA多因子驗證（TOTP）

標準 RFC 6238 時間動態碼，與 Google Authenticator 等驗證器 App 相容。

①

綁定

登入後啟動綁定，取得 QR Code 用驗證器 App 掃描；此時尚未生效。

②

啟用

輸入 App 上的第一組 6 位數完成啟用，同時取得 10 組一次性備援碼（請立刻收好，之後不再顯示）。

③

日常登入

密碼通過後多一步：輸入當下的 6 位數動態碼（允許 ±30 秒時鐘誤差）。

🆘

手機遺失

用備援碼代替動態碼登入，每組只能用一次；登入後建議重新綁定新裝置。

🔓

停用

為防盜用，關閉 MFA 需同時提供密碼＋當下動態碼重新驗證身分。

🔏

密鑰保護

TOTP 密鑰加密儲存、絕不寫入日誌；備援碼僅存雜湊。

INSIGHT監控與分析

服務內建營運儀表板：即時掌握使用量、註冊成長、活躍度與各接入平台的健康——並以隱私優先的方式記錄。

📈

分析維度

使用量、註冊、活躍度、廠商、使用分析、安全事件。

🗓️

事件保留

🔒

隱私設計

只記錄雜湊後的來源位址、不存明文，符合最小蒐集原則。

🧭

看得懂的營運指標

累計與每日新註冊、DAU／WAU／MAU 活躍度、登入成功率、有效工作階段、MFA 採用率、各平台排行——一頁掌握平台成長與健康。

🛡️

安全與濫用洞察

帳號鎖定、憑證盜用偵測、MFA 驗證失敗等趨勢即時呈現，異常一眼看見。

🏢

多平台分別檢視

可跨全平台總覽，也能鑽取單一接入平台的完整分析，資料嚴格依平台隔離。

👁️

唯讀、不碰帳號

儀表板僅提供分析，不具備任何帳號異動能力——降低後台成為攻擊目標的風險。管理者本身也受獨立空間與多因子驗證保護。
